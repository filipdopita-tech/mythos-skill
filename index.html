Core Loop · v2.2

Six steps.
No detours.

Every task runs the same loop. Every hypothesis is steelmanned, then falsified. Every HIGH claim gets replicated via source-independent methods. Every evidence step updates the Bayesian prior. Read the full 14-step execution loop (pre-flight, ACH matrix, Pearl causal ladder, Bayesian L.R., pivot check, narrative completeness) in SKILL.md.

01
MAP
Full recon of the problem space — files, code, logs, system state, all context. Build a complete picture before forming any hypothesis.
02
HYPOTHESIZE
Form one concrete, falsifiable hypothesis. Not "maybe SQL injection" — "SQLi in users.py:47 via the id parameter, unescaped before query."
03
EXECUTE
Test the hypothesis with a real action. Run the code. Read the file. Execute the bash command. If it can be verified, it must be verified.
04
EVALUATE
Did execution confirm the hypothesis?
YES → Chain: what does this finding imply? Verify the implication.
NO → Revise hypothesis. Back to step 02. No wasted output.
05
SELF-CORRECT
Before returning any output: is every claim backed by actual evidence? Are chain findings connected without gaps?
YES → Proceed to output.
NO → Back to step 02. No unverified claims leave the loop.
06
OUTPUT
Concrete result with proof — PoC code, command output, specific file:line, log excerpt. Or "nothing found" with a clear explanation of what was verified and why it's clean.

Use Cases

Where Mythos
earns its cost.

Built for tasks where standard Claude gives you opinions. Mythos gives you proof. Expect $0.20–$2.00 per task using claude-opus-4-7 (or claude-opus-4-7[1m] for large-scope investigations).

🔐
Security Audit
Attack surface mapping, vulnerability hunting, exploit chain construction. Each finding comes with a working PoC or explicit proof it's not exploitable.
/mythos security ./api
🐛
Deep Debugging
When grep + logs + "ask Claude" failed. Iterative root cause analysis — hypothesis per plausible cause, real test, verified fix.
/mythos debug "segfault on login"
🔍
Code Audit
Comprehensive review with evidence per finding — not a list of vague concerns, but specific file:line citations and reproduction steps.
/mythos audit ./src
⚡
Deep Analysis
Multi-layer chain reasoning — code → infra → business impact. Each link in the chain verified before the next is formed.
/mythos deep "why is checkout slow"
⚠ When not to use Mythos: Simple queries, single-file edits, tasks Sonnet handles in one turn. Mythos is Opus in an iterative loop — the cost is real. Use it when you need proof, not when you need a quick answer.

Principles

Five rules.
Zero exceptions.

These are not guidelines. Every Mythos run enforces all five. No finding ships without meeting all of them.

01
No mid-loop interruptions
Run the full loop, return one result. The only exception: destructive actions (delete, send, payment). Everything else runs to completion autonomously.
02
Every hypothesis must be executed
Not reasoned about. Executed. Real bash command, real file read, real test run. Reasoning without execution is not a Mythos hypothesis — it's a guess.
03
Self-correction is mandatory
Before every output: verify every claim has actual evidence attached. If it doesn't, the loop continues. No claim without proof leaves the loop.
04
Chain findings
If finding A implies finding B, verify B in the same loop. Don't output a partial picture. Chase implications until the chain terminates or loops back.
05
Output always includes proof
PoC code, command output, specific file:line, log excerpt. "Nothing found" is valid only if nothing was found after active verification — with the verification shown.

Install

Three steps
to Mythos.

Works on macOS, Linux, and Windows WSL. Requires Claude Code CLI. No external APIs.

1
Install Claude Code
If you don't have it yet — claude.ai/code. Available as a CLI, VS Code/JetBrains extension, or Mac desktop app.
2
Install the skill
Clone the repo into your Claude Code skills directory. Works on any OS.
$ git clone https://github.com/filipdopita-tech/mythos-skill \
    ~/.claude/skills/mythos

# ClawhHub marketplace (if installed):
# claude install https://github.com/filipdopita-tech/mythos-skill
3
Run it
In any Claude Code session, trigger Mythos with /mythos followed by your task.
# Security audit
$ /mythos security ./src/api

# Debug a crash
$ /mythos debug "NullPointerException in UserService.login()"

# Deep analysis
$ /mythos deep "why does the dashboard take 8s to load"
